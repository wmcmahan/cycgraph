Package-wide hardening from a full code audit, plus cross-episode entity resolution — the component that makes conflict detection and entity-based retrieval work on real multi-episode stores.

**New: `EntityResolver`** (`new EntityResolver(store).resolve()`). Extraction mints entity IDs per episode, so every cross-entity feature (all three `ConflictDetector` types, entity-BFS retrieval) was structurally inert across episodes. The resolver merges duplicates on normalized name + type (deterministic, no fuzzy matching), remaps facts' `entity_ids` and relationship endpoints to the canonical (oldest) record, drops self-loops and duplicate edges, and soft-deletes losers with a `superseded_by` forwarding pointer. Collect-then-apply, idempotent. Run it before `detectConflicts()`. `normalizeEntityName` is exported alongside.

**New: usage tracking and batched ledger stats.** `MemoryStore.touchFacts?(ids)` (optional) is called by both retrievers after serving facts; consolidation's decay scoring now ages facts from `last_accessed_at ?? valid_from`, so actively-used facts outlive untouched newer ones. `OutcomeLedger.getFactStatsBatch?(ids)` (optional) removes the N+1 in gated lesson retrieval. `LLMExtractor` accepts an injectable `logger` and reports circuit-breaker transitions.

**Extraction correctness** (`RuleBasedExtractor`): verb matching is word-boundary-anchored (no more `use` edges fabricated from "because"/"causes"); explicitly negated sentences no longer produce affirmative edges; entity coverage checks are word-level (the person "Ann" is no longer suppressed by "Annual Report"); days, months, and bare honorifics ("Dr") are no longer detected as entities — they were stealing relationship attribution from real endpoints; possessive apostrophes no longer open phantom quoted-term entities; org typing recognizes natural cues (University, Medical Center, FC, Consortium, …); 17 common news verbs added (acquire, appoint, replace, fund, sign, chair, …). Measured on an implementation-blind corpus: entity type accuracy 0.35 → 0.65, natural-text relationship capture 0.00 → 0.15.

**Contract fixes:** `retrieveMemory` now enforces the documented `QUARANTINE_TAG` invariant on all three paths (audit access via explicit `tags: ['quarantined']`); theme clusterers set `fact.theme_id` and extractors set `episode.fact_ids` (the schema's documented back-links — previously never written); `ConsolidatingThemeClusterer` merges centroids weighted by fact count (matching the schema's centroid definition and the consolidator's math); `SimpleSemanticExtractor` stamps `provenance.source: 'derived'` like the other extractors.

**Bug fixes:** stale relationship index entries on endpoint-changing upsert (returned phantom edges to BFS); `InMemoryMemoryIndex.rebuild` is atomic (a failed rebuild no longer leaves a torn half-new index) and warns explicitly when the 10K cap truncates; decay scoring no longer zeroes out schema-parsed facts (`access_count: 0`); episode pruning and conflict detection page through the whole store instead of silently capping at 10K; subgraph extraction returns a closed edge set under the entity budget; `LLMExtractor` clears its timeout timer (processes no longer hang after extraction) and resolves relationships against all declared entities regardless of item order; the retention gate never emits a duplicate `verified` tag and validates `maxTrials >= minTrials`; `gateOperatingCharacteristics` rejects `replicates > 999` (seed-stream collisions).

**Breaking — camelCase API surface.** The package now follows the workspace convention (camelCase in code, snake_case only for persisted records): every option, filter, query, policy, stats, and report shape is camelCase — `MemoryQuery` (`entityIds`, `maxHops`, `minSimilarity`, `validAt`, `changedSince`, `includeInvalidated`), store filters (`entityType`, `themeId`, `entityId`, `relationType`, `excludeTags`), `SearchOptions.minSimilarity`, `SubgraphOptions`, `TemporalFilterOptions`, `GatedLessonOptions` (`maxFacts`, `candidateSlots`, `restAfterTrials`), `RetentionPolicy` (`minTrials`, `promoteMargin`, `noiseFloorSd`, `maxBaselineRuns`, `decisionRule`, …), `RetentionReport`/`RetentionEvidence` (`factId`, `pPromote`, `baselineRuns`, …), `FactStats`/`OutcomeBaseline` (`factId`, `meanScore`), clusterer/segmenter options, `WelchLiftInput`/`Result`, and the gate-simulator config/result types. The seven persisted record schemas (`Entity`, `Relationship`, `Message`, `Episode`, `SemanticFact`, `Theme`, `Provenance`) and `RunOutcome` keep snake_case fields — they are stored rows, per the workspace's serialization-boundary rule.

**Breaking:** `EmbeddingProvider` is now shape-aligned with `@cycgraph/context-engine` — batch-only `embed(texts: string[]): Promise<number[][]>`; `embedBatch` removed (nothing in this package called the interface internally).
